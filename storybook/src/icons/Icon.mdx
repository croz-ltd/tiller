# Icon

The `Icon` component is used to easily display an arbitrary icon.

For each `Icon` component, Tiller uses **Phosphor** icon family.

You can search the full list of these icons [here](https://phosphoricons.com/).

All props are described in detail on the [Icon Props](#icon-props) section of the documentation.

Since Phosphor icons used are of font type and not SVG, the size of the individual icon is defined by `font size` CSS property. `Size` prop allows for usage of certain predefined sizes based on tailwinds default font size classes:

```json
{
  1: "text-xs",
  2: "text-sm",
  3: "text-base",
  4: "text-lg",
  5: "text-xl", (default size prop)
  6: "text-2xl",
  7: "text-3xl",
  8: "text-4xl",
  9: "text-5xl",
  10: "text-6xl",
  11: "text-7xl",
  12: "text-8xl",
  13: "text-9xl",
}
```

If there's a need for an arbitrary value so you can define a certain uncommon icon size - use `style` prop and define a custom font size.

## Using custom Icon library

In case you don't like the default icon configuration, you can easily adjust it depending on your needs. If you created the project using [tiller-start-cra](https://github.com/croz-ltd/tiller-start-cra) or [tiller-starter-vite](https://github.com/croz-ltd/tiller-starter-vite), you have a `tillerConfig.tsx` file by default.
Inside, the icon configuration is defined as
```tsx
export const defaultIconConfig: IconConfig = {
  ...iconConfig,
};
```

Here you can specify your customized icons, for example:
```tsx
  openExpander: (props: Partial<IconProps>) => (
    <span className={props.className}>-</span>
  ),
  closeExpander: (props: Partial<IconProps>) => (
    <span className={props.className}>+</span>
  ),
```
This `defaultIconConfig` is passed in `ThemeProvider` inside `index.tsx` file.
