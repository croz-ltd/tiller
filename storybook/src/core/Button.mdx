# Button

The `Button` component has 5 possible sizes - **`xs, sm, md, lg and xl`**.

When the `size` parameter is not explicitly stated, the component defaults to **`md`** size.

Its 3 variants - **`Filled`**, **`Outlined`** and **`Text`** - can be defined by
desired colors.

Other props are described in detail on the [Button Props section](#button-props)
of the documentation.

You can customize the `Button` component in the [Button Factory](#button-factory).

<br />

## Leading / Trailing Icon

The optional `Leading` / `Trailing` icon can be set for the `Button`.<br />
You can change whether you want the leading or the trailing icon in the `Controls`
of the **Button Factory** story in Storybook.<br /> The icon color changes according to text color, as demonstrated
below:

## Custom Via Tokens

If you want a custom button use `tokens` and/or `className`.

If you want to create a one-off custom button using tokens, pass the desired changed tokens which will apply
only to an instance of button to which you pass these tokens via `tokens` prop.

If you pass an empty object ({}) as `tokens` the button will have no styles, so you can create your arbitrary button using
`className`.

## Button Factory

The `Button Factory` story is used to easily create a button, giving you the
ability to modify most of the props and make the button of your choice by
using `Controls`. <br />

To create your custom button: <br />

- switch to the `Canvas` section of the Storybook and navigate to `Button Factory` story
  of the Button
- modify a wide array of options which translate to enabled (and modified) or disabled props on code view
- when you're happy with what you've created, switch back to the `Docs` section, and you will automatically be
  navigated to your created button
- after you click the `Show code` button below your custom button, the code of the button will be generated
